## 概要

`alexa-agent` は、Alexa デバイスを音声インターフェースとして LLM エージェントと対話できる
Alexa カスタムスキルである。ユーザーは「**Alexa、エージェンツ**」でスキルを起動し、
発話内容を Amazon Bedrock AgentCore 上のエージェントが処理して音声で応答する。

## 背景・目的

- Alexa 標準の応答は定型的であり、自由度の高い対話や複雑な依頼に対応できない
- LLM エージェントを音声インターフェースに接続することで、ハンズフリーで
  自然言語による質問応答・タスク依頼ができる体験を提供する
- まずは**ミニマルな実装(自由対話のみ)**で音声 × エージェントの体験を検証し、
  段階的にツール実行・記憶などの能力を拡張する

## 仕様(確定事項)

### プロダクトの基本動作

1. ユーザーが「Alexa、エージェンツ(を開いて)」と発話し、スキルを起動する
2. ユーザーの発話テキストがスキルエンドポイント経由で Bedrock AgentCore に渡される
3. AgentCore 上のエージェントが応答を生成し、Alexa が音声で読み上げる
4. セッションが継続している間はマルチターンで対話できる

### MVP スコープ(ミニマル実装)

| 項目 | 内容 |
| --- | --- |
| 自由対話 | ユーザー発話をそのままエージェントに渡し、応答を音声で返す |
| セッション内マルチターン | 同一スキルセッション内での文脈維持 |

### 将来ロードマップ(MVP 外)

| フェーズ | 能力 | 概要 |
| --- | --- | --- |
| Phase 2 | ツール実行(外部連携) | 天気・カレンダー等の外部 API をエージェントのツールとして呼び出す |
| Phase 3 | 記憶・パーソナライズ | ユーザーごとの会話履歴・好みを長期記憶し応答に反映する(AgentCore Memory) |

### 非スコープ

- Alexa 以外の音声デバイス・チャット UI への対応
- スキル内課金(ISP)
- 音声データそのものの処理(音声認識/合成は Alexa プラットフォームに委ねる)

### 成功基準(候補)

- 起動から応答までの体験が破綻しない(タイムアウトによるエラー終了が発生しない)
- 定型応答ではない、文脈に沿った自然な応答が返る
- マルチターン対話が成立する

## 用語集

| 用語 | 説明 |
| --- | --- |
| ASK (Alexa Skills Kit) | Alexa スキルを開発するための SDK・API 群 |
| Invocation Name | スキル起動名。「Alexa、◯◯を開いて」の ◯◯ 部分。本スキルでは「エージェンツ」 |
| Intent | Alexa 対話モデルにおけるユーザー意図の分類単位 |
| Slot | Intent に付随する可変パラメータ(発話内容の抽出に使う) |
| Bedrock AgentCore | AWS のエージェント実行基盤。Runtime / Memory / Gateway / Identity 等で構成される |
| AgentCore Runtime | エージェントロジックをホストする実行環境 |
| AgentCore Identity | エージェントの認証認可・外部サービス資格情報を管理する AgentCore の機能 |
| Amazon Cognito | Phase 3のAlexa Account Linkingで使用する認証基盤。MVPのRuntime認証には使用しない |
| Mastra | TypeScript 製のエージェントフレームワーク。エージェントロジックの実装に使用 |
